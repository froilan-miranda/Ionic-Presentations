Mobile Hybrid Architecture

Almost every mobile operating system (also called Platform when working with Cordova) has an API to develop apps.

This API consists of a component named Web View.

Mobile Hybrid Architecture

You can use the same knowledge of web development to build native-hybrid mobile apps

  • Android uses Android Application Package (.apk)
  • iOS uses iPhone Application Archive (.ipa)
  • Windows Phone uses Application Package (.xap)

Mobile Hybrid Architecture

This setup of showing a web page inside the mobile app container that consists of your application business logic is called as a Hybrid App.

Apache Cordova

Cordova is the piece of software that stitches the web application and the native application together

"Apache Cordova is a platform for building native mobile applications using HTML, CSS and JavaScript."

Apache Cordova

Apache Cordova also provides a set of APIs written in JavaScript to interact with the native features of the device

Apache Cordova

Cordova has a bunch of APIs that interface with the web view using JavaScript and then talk to the device in its native language (for example, Java for Android), thus providing a bridge between them

Apache Cordova

To know more from the JavaScript about the device in which your app is running, write the following code inside the JS file: 


var platform = device.platform;

Apache Cordova

To access the UUID, model, OS version, and the Cordova version of the device from inside the web view using JavaScript 


var uuid = device.uuid;
var model = device.model;
var version = device.version;
var Cordova = device.Cordova;

Apache Cordova

Cordova does not convert the HTML, CSS, and JS code to an OS-specific binary code. All it does is wrap the HTML, CSS, and JS code and execute it inside a web view.

What is Ionic?

Ionic is an open source, front-end SDK for developing hybrid mobile apps with HTML5

What is Ionic?

With the power of AngularJS inside a framework like Ionic, the possibilities are unlimited (you can use any AngularJS component inside Ionic as long as it makes sense in a mobile app)

Hello Ionic

Ionic has three main/go-to templates using which we can quickly start developing apps:

  • Blank: This is a blank Ionic project with one page
  • Tabs: This is a sample app that is built using Ionic tabs
  • Side menu: This is a sample app that is built to consume side menu driven navigation

Hello Ionic


ionic start -a "Example 1" -i app.example.one example1 blank

  • -a "Example 1": This is the human-readable name of the app
  • –i app.example.one: This is the app ID/reverse domain name
  • example1: This is the name of the folder
  • blank: This is the name of the template

Hello Ionic

cd command into the new folder and run the following:

ionic serve -p 8200

Project Structure

bower.json: This consists of the dependencies loaded via Bower. In future, we will install other Bower packages to be used along with our app. All the information regarding the same will be available here.

Project Structure

config.xml: This file consists of all the meta information needed by Cordova while converting your Ionic app to a platform-specific installer. If you open config.xml, you will see a bunch of XML tags that describes your project. We will take a look at this file in detail again.

Project Structure

gulpfile.js: This file consists of the build tasks that we would be using while developing the Ionic app.

Project Structure

ionic.project: This file consists of the information regarding the Ionic app.

Project Structure

hooks: This folder consists of scripts that get executed when a particular Cordova task is performed.

Project Structure

plugins: This folder consists of all the plugins added to the project. We will be adding a few more plugins later on, and you can see them reflected here.

Project Structure

scss: This folder consists of the base scss file which we will be overwriting to customize the styles of the Ionic components.

Project Structure

www: This folder consists of the Ionic code. Anything you write inside this folder is intended to land inside the web view. This is where we will be spending most of our time.

Project Structure

The config.xml file

This file consists of all the information needed by Cordova to convert the code in the www folder to the platform-specific installer.

There are two types of configurations that you can add to this file. One is global, that is, common to all devices, and the other is specific to the platform.

Project Structure

The www folder

This folder consists of our Ionic app, the HTML, CSS, and JS codes

There are two types of configurations that you can add to this file. One is global, that is, common to all devices, and the other is specific to the platform.

Importing Template URLs

If you want to, instead of typing the name of the template, like, say, sideMenu, you can give it the URL to the GitHub page

Importing Template URLs

The Ionic start command works by passing it along just a GitHub address instead of a named template.

And the named template are actually GitHub shortcuts.

Importing Template URLs


ionic start someProject sidemenu

Is the same as: 


ionic start someProject https://github.com/driftyco/ionic-starter-sidemenu
